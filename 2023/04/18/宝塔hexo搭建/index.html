发表于 2023-04-18 更新于 2024-06-04 分类于 博客

这是一篇在宝塔服务器部署并设置hexo-admin的文章

引言

为什么花了两天时间来搭建这个博客呢，起初对外口径是为了学习、读论文做笔记来做这个博客，其实并不是这样，我也深知这个博客也许对我的论文可能没有任何帮助，只不过是给花费时间做“没有意义”的事情一个合理的借口罢了。

那究其原因到底是为什么呢？说到底就是感兴趣，哪怕有现成的幕布、CSDN、小红书等等笔记软件，仍然还是想自己动手做一个。其实这也是我的写代码不解的渊源。

虽然大一大二都有开设计算机相关课程，但是都是一段一段零碎的练习，真正让我眼睛一亮的就是大三疫情在家，以搭建一个自己的博客为起点的前端学习，当时各种网页特效、三维GIS让我仿佛跨越了时空，因此这个博客搭建可以说是我真正喜欢上写代码的开始。

时隔两年后，我又重新再构建一遍，第一次花了两个月，把所有知道的功能全部放到博客：打字特效、音乐、评论等等，现在反而知道了一个博客的内容质量胜过这些花里胡哨的功能（可能是看到了最近的ChatGPT还有一些AI界面，虽然丑但是确实也是爱不释手，反而有种返璞归真的感觉），这次博客搭建的核心就是优化博客发布和阅读体验，让这个项目真正落地、运用到生活中，而不是一个花瓶。

当我的网页书签越来越多，我知道，是时候开始写博客了……

当然，由于能力有限，这个搭建的过程也会不断丰富。

服务器初始化

宝塔界面用nginx对域名跳转进行限制

在宝塔界面的软件商店中安装：Node.js版本管理器 1.9工具，并在该插件设置中安装nodev16.20.0（需要更新列表）

来到文件版块，创建blog文件夹，并在该文件夹目录下打开终端，从github上拉取hexo项目，注意，从2021年8月13日，git不支持在https模式下输入用户名和密码来下载代码，因此需要配置ssh或者在github中创建token

创建token方法：进入官网，点击头像进入settings，然后选择左侧最下方的Developer settings，在左侧选择Personal access tokens，并在转开列表中选择：Tokens（classic），然后在右上角选择Generate new token，打开所有选项功能，获得Token（注意保存，只显示一次）

在拉取代码之前，设置邮箱和用户名，拉取地址变为：https://Token@github.com/vector9584/仓库名称

下载好代码之后开始部署hexo环境

  • npm install hexo -g
  • npm install hexo-cli -g
  • ln -s /www/server/nodejs/v16.20.0/lib/node_modules/hexo/bin/hexo /usr/bin/hexo

配置好环境之后就可以部署项目了，首先安装themes（上传代码时，themes也是git拉取下来的，不能同步上去，因此需要单独安装）

  • git clone https://github.com/next-theme/hexo-theme-next themes/next

配置启动脚本：在blog文件夹下，找到package.json，在scripts中添加："dd": "hexo clean && hexo g && hexo s"

然后宝塔的网站面板中进入Node项目选项卡，添加node项目，选择blog文件夹，然后在下方进入设置

在项目配置中，启动选项选择dd，运行用户选择root，在模块管理中一键安装项目模块，然后对项目进行启动

环境配置

通过在deploy脚本中添加github同步代码，实现文章同步的同时，备份仓库代码，但是git代码需要写在后面，loading时间比较长，不用等待结果。

发现这样每次要等很长时间，还是先删除了，隔一段时间备份一下就行了。

当设置hexo-admin账户密码后，在页面登录之后，宝塔的node自动停止，需要重新启动提交表单才能访问，因此不得不把登陆功能关了，后期可以自己设计一个登陆页面。
