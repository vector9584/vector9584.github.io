未来一年测科院工作计划

23-06-07

在windows电脑上部署了环境，有以下需要注意的点

构建项目流程

  • 项目地址
  • 下载nvm（路径不能有中文），安装12.12.0，并切换该环境
  • 部署gitee公匙，拉取代码
  • windows安装redis
    • https://www.runoob.com/redis/redis-install.html
    • redis启动出错
      • https://www.cnblogs.com/shaosks/p/7089786.html
  • 修改代码环境
    • 修改maven为默认环境3.8.1
    • 设置node环境为12.12.0（nvm use 12.12.0）
    • 设置日志目录
    • 设置文件目录

23-04-25

今天部署了程序代码

  • 拉取代码：http://192.168.210.237:8090/casm/bigdata-management.git
  • 通过meven安装依赖
  • 修改host端口号
  • mac安装redis
  • 查看版本
  • npm 6.11.3
  • node v12.12.0
  • 多个版本管理方法：https://www.runoob.com/w3cnote/nvm-manager-node-versions.html
  • 部署项目
  • 后台：CasmApplication
  • 前端：bigdata-ui

23-04-01

数据管理

  • 数据添加类型抽象，需要扩展能力（tif、b3dm、表格、postGIS、mongodb：瓦片，特殊的非关系性数据，先不考虑，不太好管理入库过程，可以外挂）
  • 接口解析
  • 流数据管理（sqlServer定时更新，数据同步）
  • 矿山数据平台管理
  • 提供给应急部
  • 先实现以前的功能
  • Wms结构化数据表格数据发布，根据服务模块和解析形成服务，配图用Geowinmap
  • GeoServer也用其中的一些包（GeoTools）
  • 目前数据库用：PostGIS
  • 其他数据库存面状数据用wkt格式
  • 在中间层SDK转成geomatch
  • 三维数据：Dem、模型等属于瓦片服务：wtms
  • 每个服务都是独立的（展示可以使用图层组）
  • 文本、word、pdf等下载、浏览
  • 现在的数据是针对数据
  • 数据专题化在前端进行组织
  • 可以考虑加入一些Readis
  • 点保存要调用Geomap服务，要清缓存，重新读记录进行更新
  • 数据源
    • 数据目录
      • 数据服务
        • 表格数据服务
          • 可视化：统计图表
          • 服务注册
  • 瓦片类、非结构化等，数据库移动等用相对路径，只需要改根目录，大概率单机运行
  • 脚本化、可执行化、cesium等，cs往web框架靠，bs在node里面跑
  • Bat、exe（config描述：输入参数，如文件、下拉、动态计算）->缓存
  • geoconfig、maplink
  • 通过需求开发服务
  • 多层数据调用要考虑缓存，生成缓存表，添加消息中间件，否则调用频繁
  • 对外路径一样，后台用Nginx包装
  • 五月底、六月份要初稿
  • Geowinmap修改，分发成微服务
  • 高效路由，读数据库缓存等比较麻烦
  • navicat，6w
  • 第三方数据集成、相互触发更新
  • 服务ID访问参数读取
  • RPC阿里消息服务中间价，springboot
  • readis
  • 院里做一个平台（一张图）900w
