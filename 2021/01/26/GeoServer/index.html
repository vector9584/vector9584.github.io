GeoServer

发表于 2021-01-26 更新于 2024-11-11

GeoServer is an open source server for sharing geospatial data. Designed for interoperability, it publishes data from any major spatial data source using open standards.

golang:linux系统下集成了许多开源GIS工具的Linux

geoserver和arcgis server功能类似：实现地图发布功能、分享地图

GeoServer:Open source server for sharing geospatial data

商用GIS工具能实现的，用开源工具同样可以实现

环境准备

Java环境（1.8）

安装包下载

java8下载地址

配置Java Path

打开环境变量

  • 方法1：选中我的电脑 --> 键盘按下 Alt + Enter --> 高级系统设置 --> 环境变量

  • 方法2：键盘按下 win + r --> 输入 sysdm.cpl --> 高级 --> 环境变量

  • 方法3：win10系统下，工具栏有搜索图标，点击后直接搜索"环境变量"

添加用户变量 —— JAVA_HOME

点击新建，输入变量名(JAVA_HOME)与变量值(java安装路径Java1.8.0)，点击确定。

添加用户变量 —— CLASSPATH

继续点击新建，输入变量名与变量值，点击确定

变量值：;%JAVA_HOME%\lib;%JAVA_HOME%\lib\tools.jar;

编辑用户变量 —— Path

选中Path，点击编辑，点击新建，输入下面的路径，记得一行输入一个路径。

%JAVA_HOME%\bin

%JAVA_HOME%\jre\bin

所有的窗口一路按下【确定】，否则无效。

验证试运行

方法：键盘按下 win + r --> 输入 cmd --> 键盘按下 回车 --> 输入java --> 键盘按下回车

结果：若出现一堆提示性语句，则表示安装成功。

在cmd下，也可以输入 java -version 试试。

Tomcat（8）

下载官网

下载压缩包并解压

进入bin目录

复制地址，添加到环境变量

bat文件是windows下运行的批处理文件

sh是Linux系统下运行的批处理文件

把tomcat的编码修改和系统一致就解决乱码问题，方法入戏：

1、找到apache-tomcat-7.0.92/conf/logging.properties

2、添加语句：java.util.logging.ConsoleHandler.encoding = GBK

GeoServer（war 2.16）

下载官网

选择Web Archive（Web Archive (war) for servlet containers.）下载war包

war包：前端页面集成包

将下载的war.zip文件进行解压

将war文件放入Tomcat安装路径下webapps中

涉及到的插件（GeoServer Plugin）

  • mbtiles(2.16)

mb-tiles是一类数据格式

  • wps(2.16)

是运行mbtiles的一个环境

QGIS（3.14）

涉及到的插件（QGIS Plugin）

  • SLD4raster：支持RGB三波段显示

其他

  • CSS：层叠样式表
  • PostGIS：数据库（基于Postgrid进行的二次开发）
  • Udig：为GeoServer配图的工具
  • Bulk vector export as SLD and GeoJson：QGIS插件

运行

在Tomcat的bin目录中找到startup.bat运行就可以了

此过程中会自动解压war包

注：startup调用的是同目录下catalina.sh

启动后访问localhost:8080/geoserver

初始账号：admin

初始密码：geoserver

矢量数据发布与样式联动

数据发布

首先进入工作区目录，添加新的工作区

Name：WMS

命名空间URI：wms.lol

点击保存

进入启用，并作为默认工作区

最后保存

进入数据存储目录

添加新的数据存储

选择文件格式（shp）

选择工作区（此时是默认工作空间，上一步设置的）

数据源名称成（line）

文件位置（点选发布文件）

文件夹方式：将发布文件放到适合GeoServer文件选择的位置

数据库方式：调用数据库进行操作

字符集（矢量数据table中的编码方式，要与发布时一致）：UTF-8

点保存之后图层就已经保存了

图层的发布

继续刚刚的页面，点击发布

图层名称设置

数据有很多级别，最后有一个明确的记录：哪一层用了哪个名字

边框

从数据中计算->Compute from native bounds

此处会计算数据边界并给出对应位置

在下面的要素类型中是dbf属性信息，从这里我们也可以确认之前的字符集编码格式是否选择正确

保存

然后进入Layer Preview

在openlayers中可以查看（OpenLyers）

样式联动

进入Styles

Add a new stye

有两种实现格式的方法：SLD（OGC认可的方式）、装一个CSS插件

输入名称

选择工作区

Format选择SLD

Generate a default style选择Line（shp格式为线状）

然后点击后面的Generate，将会在下方生成样式模板

点击最下面的Validate进行验证，看看是否有语法错误（其实写错了也没关系）

当点击Apply（应用）之后上方就会多出一些功能：data;publishing;layer preview;layer attributes

在Publishing模块里面可以对添加的图层进行样式链接（Associated打勾）

点击Apply进行保存

再进入Layer preview就可以预览，在右上角还有图例

到此为止大家就尝试了SLD修改样式，但是大家会发现，修改样式居然用代码来进行设置，不够优雅

接下来我们用QGIS解决

首先打开QGIS，加载该文件

然后调整样式

图层>属性>渲染

可以再里面设置在多少比例尺出现，在多少比例尺消失

符号化>基于规则

双击进入，进行基于样式的显示设置（和上方功能一样，但是是基于样式的）

按住ctrl可以缓慢缩放

设置完成后，Apply

属性页面左下角

样式->保存样式

新打开对话框中保存样式选择：作为SLD样式文件

然后选择文件保存路径

回到我们的GeoServer样式管理界面，在Data标签中找到Style Content

单机选择文件，导入之前设置的样式文件（sld文件）

然后点击Upload，覆盖当前的样式

此时观察最下面的代码，已经改变了

Apply之后我们在Layer Preview就可以看到修改后的样式

打开QGIS，点击菜单栏的插件->管理并安装插件

搜索并安装SLD4raster

加载栅格文件

打开属性->符号化

此处我们选择单波段伪彩色

进行基本的样式设置

然后打开插件进行基本设置

进行上传

回到GeoServer，在style里面我们可以看到新添加了一个样式

透明度

将这些文件剪切出来，放在Tomcat文件夹下的
